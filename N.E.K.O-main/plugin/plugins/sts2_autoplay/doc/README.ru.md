# Быстрый старт

`sts2_autoplay` используется для подключения в N.E.K.O локального состояния *Slay the Spire 2*, которое публикует `STS2 AI Agent`. Плагин может читать текущую ситуацию, выполнять допустимые действия, автоматически играть по стратегии, позволять кошкодевочке выбирать одну карту, отправлять информацию наблюдения на фронтенд, а также позволять кошкодевочке отправлять мягкие подсказки в фоновой задаче, чтобы влиять на следующий раунд принятия решений.

## Руководство по использованию

### Получение MOD

Используя Git:
```text
https://github.com/CharTyr/STS2-Agent/releases
```

### Установка игрового Mod

В Steam нажмите правой кнопкой мыши по *Slay the Spire 2* и выберите Управление -> Просмотреть локальные файлы.

Стандартный каталог игры Steam обычно выглядит примерно так:

```text
...\Steam\steamapps\common\Slay the Spire 2
```

Скопируйте mod `STS2 AI Agent` в каталог `mods/` внутри папки игры.

Если в каталоге *Slay the Spire 2* нет папки `mods`, создайте её вручную.

```text
Использование модов может привести к потере сохранений. Сделайте резервную копию или компенсируйте это через консоль (в главном меню Slay the Spire нажмите клавишу "~", введите "unlock all", и все персонажи и уровни сложности будут разблокированы).
```

После установки структура каталога должна быть примерно такой:

```text
Slay the Spire 2/
  mods/
    STS2AIAgent.dll
    STS2AIAgent.pck
    mod_id.json
```

### Запуск игры и проверка интерфейса

Сначала запустите игру обычным способом, чтобы Mod загрузился вместе с ней.

При первом переключении в режим с модом игра может один раз вылететь. Это нормально — просто запустите игру снова.

После загрузки мода в N.E.K.O включите Cat Paw, активируйте плагин, откройте панель плагина и вручную запустите плагин Slay the Spire.

### Доступные команды

【Сыграй карту】【Автоиграй за меня】【Пройди этаж】【Как я сыграл】【Стоп】
【Сыграй одну карту】【Сыграй определённую карту】【Порекомендуй карту】…… и подобные фразы.

## Обзор возможностей

- Подключается к локальному HTTP-сервису `STS2 AI Agent` и считывает текущее состояние забега.
- Позволяет одним действием посмотреть текущую ситуацию: один раз обновляет состояние и возвращает вместе снимок, краткое описание ситуации и пакет синхронизации неко.
- Позволяет управлять autoplay в фоне: запускать, ставить на паузу, возобновлять, останавливать, а также сразу выполнять следующий предложенный шаг.
- Поддерживает режим сопровождения: он может наблюдать за забегом вместе с вами и отправлять комментарии, напоминания и наблюдения, не ломая основной поток.
- Поддерживает настройку стратегии на естественном языке: одна фраза пользователя может быть превращена в override предпочтений для текущего события или врага.
- Позволяет заранее посмотреть, какой ход предлагается следующим, прежде чем решать, выполнять его или нет.
- Включает защитные механизмы: паузу при низком HP, замедление при опасных атаках, восстановление скорости после снятия угрозы и, когда это безопасно, возобновление autoplay.
- Также поддерживает пассивные отправки во фронтенд: синхронизацию состояния, наблюдения, подсказки режима сопровождения и служебную обратную связь по управлению.

## Конфигурация плагина

Файл конфигурации: `plugin.toml`

### Базовая конфигурация

| Параметр | Значение по умолчанию | Описание |
| --- | --- | --- |
| `base_url` | `http://127.0.0.1:8080` | Адрес локального Agent для Spire. |
| `connect_timeout_seconds` | `5` | Таймаут подключения в секундах. |
| `request_timeout_seconds` | `15` | Таймаут запроса в секундах. |
| `poll_interval_idle_seconds` | `3` | Интервал опроса в состоянии ожидания. |
| `poll_interval_active_seconds` | `1` | Интервал опроса во время работы autoplay. |
| `action_interval_seconds` | `1.5` | Дополнительная пауза между действиями. |
| `post_action_delay_seconds` | `0.5` | Задержка после действия, чтобы дождаться стабилизации ситуации. |
| `autoplay_on_start` | `false` | Нужно ли автоматически запускать autoplay при старте плагина. |
| `character_strategy` | `defect` | Стратегия по умолчанию; во время выполнения она сопоставляется с наиболее подходящим стратегическим контекстом для текущей ситуации. |
| `max_consecutive_errors` | `3` | Максимальное число последовательных ошибок, после которого соединение считается проблемным. |

### Отправка во фронтенд и наблюдение режима сопровождения

| Параметр | Значение по умолчанию | Описание |
| --- | --- | --- |
| `llm_frontend_output_enabled` | `true` | Можно ли отправлять во фронтенд действия и ошибки autoplay. |
| `llm_frontend_output_probability` | `1.0` | Вероятность отправки обычных сообщений о действиях. Ошибки и некоторые важные сигналы управления всё ещё могут отправляться принудительно. |
| `autoplay_push_probability` | `0.5` | Вероятность отправки обычной синхронизации состояния, когда режим сопровождения не активен. |
| `companion_push_probability` | `0.7` | Вероятность отправки обычной синхронизации состояния, когда режим сопровождения активен. |
| `neko_reporting_enabled` | `true` | Включать ли наблюдательную механику неко. |
| `neko_report_interval_steps` | `1` | Через сколько шагов autoplay пересобирать блок наблюдений. |
| `neko_report_hud_enabled` | `true` | Отправлять ли подготовленный блок наблюдений в HUD / канал сообщений фронтенда. |
| `neko_commentary_enabled` | `true` | Разрешено ли генерировать комментарии и напоминания режима сопровождения. |
| `neko_commentary_probability` | `0.65` | Вероятность срабатывания обычных комментариев низкого приоритета. |
| `neko_commentary_min_interval_seconds` | `4` | Минимальный интервал перед повтором похожих комментариев; нужен для снижения спама. |
| `neko_critical_commentary_always` | `true` | Нужно ли всегда озвучивать предупреждения высокого приоритета. |
| `neko_guidance_max_queue` | `50` | Внутренний лимит очереди для контекста, связанного с подсказками и предпочтениями. |

### Автоматическая защита и контроль темпа

| Параметр | Значение по умолчанию | Описание |
| --- | --- | --- |
| `neko_auto_low_hp_threshold` | `0.3` | Если доля HP падает ниже этого значения, autoplay предпочитает поставить игру на паузу. |
| `neko_auto_safe_hp_threshold` | `0.5` | Когда HP возвращается в этот диапазон, ситуация снова может считаться безопасной. |
| `neko_auto_dangerous_attack_threshold` | `20` | Если вражеское намерение наносит большой урон и достигает этого порога, может сработать защитное замедление. |
| `neko_auto_resume_after_low_hp` | `true` | Разрешать ли автоматическое возобновление после паузы на низком HP, когда ситуация снова стала безопасной. |
| `neko_desperate_enabled` | `true` | Включать ли стойку выживания на грани смерти. |
| `neko_desperate_hp_threshold` | `0.2` | Порог HP, при котором включается эта стойка выживания. |
| `neko_maximize_enabled` | `true` | Включать ли более выраженную ориентацию на максимизацию выгоды. |

## Рекомендуемые формулировки для обычных пользователей

Обычным пользователям не нужно запоминать низкоуровневые параметры. Проще всего передавать фразу пользователя в оставшиеся высокоуровневые точки входа и позволять плагину самому решать, идёт ли речь о просмотре ситуации, корректировке стратегии или выполнении следующего предложенного шага.

Рекомендуемая трактовка:

| Что хочет сказать пользователь | Какая возможность подходит лучше |
| --- | --- |
| `что сейчас происходит` | `sts2_get_status` |
| `покажи текущую ситуацию` | `sts2_read_state` |
| `пусть она продолжит играть` / `пока останови autoplay` / `пусть снова продолжит` / `больше не нужно, чтобы она играла сама` | точки входа управления autoplay |
| `скорректируй стратегию так: в этом событии я предпочитаю более дешёвый путь` | `sts2_apply_user_override` |
| `покажи, что она хочет сделать дальше` | `sts2_get_planned_operation` |
| `сделай предложенный шаг` | `sts2_execute_planned_operation` |
| `включи режим сопровождения` / `выключи режим сопровождения` | точки входа управления режимом сопровождения |

Рекомендуемый порядок такой:
1. Сначала посмотреть текущую ситуацию.
2. Потом посмотреть, что она собирается сделать дальше.
3. Если хочется изменить приоритеты — скорректировать стратегию одной фразой.
4. В конце решить, выполнять этот шаг или позволить autoplay идти дальше.

## Точки входа плагина

Это публичные возможности, которые действительно остались во внешнем интерфейсе текущего основного скрипта. Видимые названия сделаны более естественными, но внутренние `entry id` сохраняются неизменными ради совместимости с интеграциями хоста.

### `sts2_health_check`

Проверяет, доступен ли локальный сервис Agent для Spire. Это хороший первый шаг при запуске, интеграции или разборе ошибок.

### `sts2_get_status`

Показывает общее состояние рантайма: здорова ли связь, на каком экране сейчас находится забег, работает ли autoplay, включён ли standby, и в каком состоянии находятся текущий режим и недавние ошибки.

### `sts2_read_state`

Один раз обновляет текущую ситуацию и возвращает сразу три слоя информации:
- текущий снимок
- текущее краткое описание ситуации
- текущий пакет синхронизации неко

Это удобно, когда нужно увидеть всю картину сразу, прежде чем решать следующий ход.

### `sts2_set_standby`

Переключает режим standby. В standby действия больше не выполняются, но способность упорядочивать состояние и готовить синхронизацию сохраняется.

### `sts2_start_autoplay`

Позволяет ей продолжить играть. Запускает autoplay в фоне и даёт текущей ситуации развиваться дальше самостоятельно.

### `sts2_pause_autoplay`

Пока ставит autoplay на паузу. Полезно, если вы хотите вмешаться вручную или поменять стратегию перед следующим действием.

### `sts2_resume_autoplay`

Заставляет её продолжить играть с того места, где autoplay был поставлен на паузу.

### `sts2_stop_autoplay`

Больше не позволяет ей играть самостоятельно. Полностью останавливает autoplay в фоне и возвращает управление вам.

### `sts2_enable_companion_mode`

Включает режим сопровождения. С ним плагин начинает чаще упорядочивать состояние, а также присылать наблюдения, комментарии и напоминания, когда это уместно.

### `sts2_disable_companion_mode`

Выключает режим сопровождения. Он убирает только слой комментариев, но оставляет базовое чтение состояния и управление autoplay.

### `sts2_apply_user_override`

Корректирует стратегию по одной пользовательской заметке. Он интерпретирует вашу фразу в контексте текущей сцены и превращает её в override для соответствующего события или врага.

У этой точки входа также есть дополнительная защита:
- если autoplay уже работает, **сначала он будет поставлен на паузу**
- после обновления стратегии плагин подскажет, что **для продолжения autoplay нужно возобновить вручную**
- он не продолжит забег сам, пока вы явно этого не решите

### `sts2_get_planned_operation`

Показывает, что она хочет сделать дальше. Подходит, если вы хотите сначала посмотреть следующую рекомендацию, а не выполнять её сразу.

### `sts2_execute_planned_operation`

Сразу выполняет следующий предложенный шаг.

## События, отправляемые во фронтенд

Плагин передаёт по каналу сообщений хоста несколько видов пассивной информации, в основном сгруппированных в три блока:

1. **Синхронизация состояния и ситуации**
   - краткое описание текущей ситуации
   - краткое описание текущей рекомендации
   - данные синхронизации во время активного режима сопровождения

2. **Обратная связь по управлению autoplay**
   - autoplay запущен
   - поставлен на паузу / возобновлён / остановлен
   - уведомление о необходимости вручную возобновить autoplay после обновления стратегии

3. **Сигналы сопровождения и защиты**
   - комментарии режима сопровождения
   - предупреждения о риске
   - пауза при низком HP
   - замедление из-за опасной атаки
   - восстановление скорости или возобновление autoplay после снятия угрозы

Эти сообщения по умолчанию используют пассивную семантику доставки и не должны насильно прерывать основной разговор. Частота их появления также зависит от настроек:
- `autoplay_push_probability`
- `companion_push_probability`
- `neko_commentary_probability`
- `neko_report_hud_enabled`

## Решение частых проблем

### Ошибка соединения при вызове точки входа плагина

Сначала проверьте:

- запущена ли игра
- лежит ли mod `STS2 AI Agent` в `mods/`
- доступен ли `http://127.0.0.1:8080/health`
- корректен ли `base_url` в `plugin.toml`

### `http://127.0.0.1:8080/health` не открывается

Проверьте по порядку:

1. действительно ли запущена игра
2. скопированы ли `STS2AIAgent.dll`, `STS2AIAgent.pck` и `mod_id.json` в `mods/`
3. не были ли файлы переименованы, продублированы или положены не в ту папку
4. работаете ли вы именно в каталоге игры Steam, а не в upstream-репозитории
5. не блокирует ли локальный порт файрвол или защитное ПО

### Autoplay работает, но фронтенд не получает сообщения

Проверьте:

- установлен ли `llm_frontend_output_enabled` в `true`
- не слишком ли низок `llm_frontend_output_probability`
- установлен ли `neko_reporting_enabled` в `true`
- во время интеграции можно временно поднять `llm_frontend_output_probability` до `1`
- действительно ли фронтенд хоста получает сообщения плагина

### Подсказка в середине забега почти не влияет на игру

Проверьте:

- не находится ли плагин сейчас в standby
- вернул ли `sts2_send_neko_guidance` статус `ok`
- достаточно ли конкретна подсказка, например `сначала защита`, `бей первым врага с наименьшим HP`, `сохрани зелье`
- позволяют ли текущие допустимые действия реально выполнить эту подсказку

### Полуавтоматическая задача не завершается

Проверьте `stop_condition`:

- если это `manual` / `none`, задача сама не завершится, и нужно вручную вызвать `sts2_stop_autoplay`
- если это `current_combat`, задача завершается после того, как в ходе её выполнения произошёл вход в бой и затем выход из него
- если это `current_floor`, обычно задача завершается после прохождения текущего этажа или перехода на следующий

Можно использовать `sts2_get_status`, чтобы посмотреть `autoplay.task`.

### Застревает в событиях, всплывающих окнах или переходных состояниях

Текущая версия уже умеет обрабатывать события, popup-окна и переходные состояния. Приоритетные действия включают:

- `confirm_modal`
- `dismiss_modal`
- `choose_event_option`
- `proceed`

Если всё равно застревает, сначала используйте `sts2_read_state`, чтобы проверить текущие `screen` и `available_actions`.

### Autoplay внезапно ставится на паузу или начинает идти медленнее

Возможно, сработала одна из защит:

- autoplay ставится на паузу, если доля HP падает ниже `neko_auto_low_hp_threshold`
- в боях с боссами или при опасных атаках он замедляется
- если `neko_auto_resume_after_low_hp` установлен в `true`, autoplay может возобновиться, когда HP вернётся к `neko_auto_safe_hp_threshold`

Можно использовать `sts2_get_status`, чтобы проверить состояние, или вызвать `sts2_resume_autoplay` / `sts2_stop_autoplay` для ручного вмешательства.
