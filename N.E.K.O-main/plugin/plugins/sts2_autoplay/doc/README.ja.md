# クイックスタート

`sts2_autoplay` は、`STS2 AI Agent` が公開しているローカルの *Slay the Spire 2* 状態を N.E.K.O に接続するためのものです。プラグインは盤面を読み取り、合法アクションを実行し、戦略に従って自動プレイし、ねこ娘に単一のカードを選ばせ、観察情報をフロントエンドにプッシュし、さらにねこ娘がバックグラウンドタスク内でソフトガイダンスを送って次の意思決定ラウンドに影響を与えることもできます。

## 使用チュートリアル

### MOD の入手

Git を使う場合：
```text
https://github.com/CharTyr/STS2-Agent/releases
```

### ゲーム Mod のインストール

Steam で *Slay the Spire 2* を右クリックし、管理 -> ローカルファイルを閲覧 を選びます。

Steam のデフォルトゲームディレクトリは通常次のような場所です：

```text
...\Steam\steamapps\common\Slay the Spire 2
```

`STS2 AI Agent` mod を、尖塔ゲームディレクトリの `mods/` にコピーしてください。

*Slay the Spire 2* のディレクトリに `mods` フォルダがない場合は、自分で作成してください。

```text
mod の使用によりセーブデータが失われる可能性があります。バックアップを取るか、コンソールで補償してください（尖塔のメインメニューで "~" キーを押し、"unlock all" と入力すると、全キャラと難易度が解放されます）。
```

インストール後のディレクトリ構成は次のようになります：

```text
Slay the Spire 2/
  mods/
    STS2AIAgent.dll
    STS2AIAgent.pck
    mod_id.json
```

### ゲームを起動してインターフェースを確認

まず通常通りゲームを起動し、Mod をゲームと一緒に読み込ませてください。

初めて mod モードに切り替えると一度クラッシュすることがありますが、正常な現象です。もう一度ゲームを起動すれば大丈夫です。

mod 読み込み後、N.E.K.O 側で Cat Paw を有効化し、プラグインを有効化し、プラグインパネルに入り、Slay the Spire プラグインを手動で起動してください。

### 使用できるコマンド

【カードを打つ】【自動で代打】【1 階を通す】【打ち方どうだった】【停止】
【1 枚カードを打つ】【特定のカードを打つ】【1 枚おすすめして】……など、これに類する表現。

## 機能概要

- ローカルの `STS2 AI Agent` HTTP サービスに接続し、現在のラン状況を読み取ります。
- 現在の状況をまとめて確認できます。状態を一度更新し、スナップショット、状況要約、ねこ娘同期パケットをまとめて返します。
- バックグラウンド自動プレイの制御に対応しています。開始・一時停止・再開・停止に加え、提案された次の一手をそのまま実行できます。
- 陪玩モードに対応しており、主な流れを遮らずに、設定に応じて観察・コメント・リマインドを送ります。
- 自然言語での戦略調整に対応しています。ユーザーの一言から、現在のイベント／敵ごとの方針オーバーライドに落とし込みます。
- 次のおすすめ行動を先に確認できます。すぐ実行する前に「次にどう動こうとしているか」を見て判断できます。
- 安全保護にも対応しています。低 HP での一時停止、危険な攻撃時の減速、危険解除後の速度復帰、必要に応じた自動プレイ再開などを行います。
- フロントエンドへの受動的な通知にも対応しています。状況同期、観察情報、陪玩ヒント、制御フィードバックはすべてプラグインのメッセージ経路を通じてホストへ送られます。

## 本プラグインの設定

設定ファイル：`plugin.toml`

### 基本設定

| 設定項目 | デフォルト値 | 説明 |
| --- | --- | --- |
| `base_url` | `http://127.0.0.1:8080` | 尖塔ローカル Agent のアドレス。 |
| `connect_timeout_seconds` | `5` | 接続タイムアウト秒数。 |
| `request_timeout_seconds` | `15` | リクエストタイムアウト秒数。 |
| `poll_interval_idle_seconds` | `3` | アイドル時のポーリング間隔。 |
| `poll_interval_active_seconds` | `1` | 自動プレイ実行中のポーリング間隔。 |
| `action_interval_seconds` | `1.5` | 各アクションの間に入る追加待機時間。 |
| `post_action_delay_seconds` | `0.5` | アクション実行後、盤面が安定するまで待つ時間。 |
| `autoplay_on_start` | `false` | プラグイン起動後に自動プレイを自動開始するか。 |
| `character_strategy` | `defect` | 既定の戦略名。実行時には現在の局面に応じて適切な戦略コンテキストへ紐づけられます。 |
| `max_consecutive_errors` | `3` | 最大連続エラー回数。これを超えると接続異常と見なします。 |

### フロントエンド通知と陪玩観察

| 設定項目 | デフォルト値 | 説明 |
| --- | --- | --- |
| `llm_frontend_output_enabled` | `true` | 自動プレイのアクション／エラーをフロントエンドへ送ってよいか。 |
| `llm_frontend_output_probability` | `1.0` | 通常アクション通知の送信確率。エラーや重要な制御フィードバックは強制送信されることがあります。 |
| `autoplay_push_probability` | `0.5` | 陪玩モード以外で、通常の局面同期を送る確率。 |
| `companion_push_probability` | `0.7` | 陪玩モード中に、通常の局面同期を送る確率。 |
| `neko_reporting_enabled` | `true` | ねこ娘の観察機能を有効にするか。 |
| `neko_report_interval_steps` | `1` | 自動プレイ何ステップごとに観察内容を整理するか。 |
| `neko_report_hud_enabled` | `true` | 整理した観察内容を実際にフロントエンド HUD／メッセージ経路へ送るか。 |
| `neko_commentary_enabled` | `true` | 陪玩コメントやリマインドを生成してよいか。 |
| `neko_commentary_probability` | `0.65` | 通常の低優先度コメントが発生する確率。 |
| `neko_commentary_min_interval_seconds` | `4` | 同種のコメントを繰り返すまでの最小間隔。スパム軽減用。 |
| `neko_critical_commentary_always` | `true` | 高優先度の注意喚起を常に流すか。 |
| `neko_guidance_max_queue` | `50` | ガイダンス／方針関連コンテキストの内部キュー上限。 |

### 自動保護とテンポ制御

| 設定項目 | デフォルト値 | 説明 |
| --- | --- | --- |
| `neko_auto_low_hp_threshold` | `0.3` | HP 比率がこの値を下回ると、自動プレイは優先的に一時停止します。 |
| `neko_auto_safe_hp_threshold` | `0.5` | HP がこの値まで戻ると、再び安全域として扱えます。 |
| `neko_auto_dangerous_attack_threshold` | `20` | 敵の高火力意図がこの閾値に達すると、減速保護が入ることがあります。 |
| `neko_auto_resume_after_low_hp` | `true` | 低 HP による一時停止後、安全条件が戻った際に自動再開を許可するか。 |
| `neko_desperate_enabled` | `true` | 瀕死時の生存重視姿勢を有効にするか。 |
| `neko_desperate_hp_threshold` | `0.2` | 瀕死生存姿勢に切り替わる HP 比率。 |
| `neko_maximize_enabled` | `true` | より価値最大化寄りの判断傾向を有効にするか。 |

## 一般ユーザー向けおすすめの言い方

一般ユーザーは低レベルのパラメータを覚える必要はありません。まずは今残っている高レベル入口にユーザーの言葉をそのまま渡し、状況確認なのか、戦略調整なのか、それとも提案された一手を実行したいのかをプラグイン側に判断させるのがおすすめです。

おすすめの捉え方：

| こう言いたいとき | 向いている機能 |
| --- | --- |
| `今どんな状況？` | `sts2_get_status` |
| `今の盤面を見せて` | `sts2_read_state` |
| `彼女にそのまま打たせて` / `いったん自動プレイを止めて` / `続きを打たせて` / `もう自動で打たなくていい` | autoplay 制御入口 |
| `このイベントは低コスト寄りで行きたい、って方針にして` | `sts2_apply_user_override` |
| `次にどう動こうとしてる？` | `sts2_get_planned_operation` |
| `その提案どおり一手進めて` | `sts2_execute_planned_operation` |
| `陪玩モードをオンにして` / `陪玩モードをオフにして` | companion mode 制御入口 |

おすすめの流れは次のとおりです：
1. まず盤面を見る
2. 次に、彼女がどう動こうとしているかを見る
3. 気になる点があれば、一言で方針を調整する
4. 最後に、そのまま一手進めるか、自動プレイを続けるかを決める

## プラグイン入口

以下は、現在の主スクリプトで実際に残っている対外機能です。表示名はできるだけ自然な言い回しに寄せていますが、内部の entry id は安定性のためそのまま維持されています。

### `sts2_health_check`

ローカルの尖塔 Agent サービスにちゃんと接続できているかを確認します。起動後の自己チェック、連携確認、エラー調査の最初の一手として向いています。

### `sts2_get_status`

現在の全体状態を確認します。接続の正常性、今どの画面にいるか、自動プレイが動いているか、standby かどうか、最近のエラーや現在モードなどをまとめて見られます。

### `sts2_read_state`

現在の局面を一度更新し、次の 3 層をまとめて返します：
- 現在のスナップショット
- 現在の状況要約
- 現在のねこ娘同期パケット

次にどうするかを決める前に、ひと通り状況をまとめて見たいときに向いています。

### `sts2_set_standby`

待機モードを切り替えます。待機中はアクション実行を止めますが、状態整理や同期準備の能力は維持されます。

### `sts2_start_autoplay`

彼女にそのまま打たせます。バックグラウンド自動プレイを起動し、現在の局面をそのまま進めます。

### `sts2_pause_autoplay`

いったん自動プレイを止めます。自分で手を入れたいときや、次の一手の前に方針を変えたいときに便利です。

### `sts2_resume_autoplay`

止めていた自動プレイをそのまま再開します。

### `sts2_stop_autoplay`

もう自動で打たせないようにします。バックグラウンド自動プレイを完全に止め、操作権をこちらに戻します。

### `sts2_enable_companion_mode`

陪玩モードをオンにします。状況整理や観察通知がより積極的になり、適切なタイミングでコメントや注意喚起も入るようになります。

### `sts2_disable_companion_mode`

陪玩モードをオフにします。陪玩コメント層だけを止め、基本的な状態取得や自動プレイ制御はそのまま使えます。

### `sts2_apply_user_override`

ユーザーの一言で戦略を調整します。現在の場面に照らして、その文をイベント単位または敵単位の方針オーバーライドに変換します。

この入口には追加の安全挙動もあります：
- 自動プレイが動作中なら、**先に自動プレイを一時停止します**
- 方針更新後は、**続けたいなら手動で自動プレイを再開するよう案内します**
- 明示的に再開されるまで、そのまま勝手に続行はしません

### `sts2_get_planned_operation`

次にどう動こうとしているかを確認します。すぐに実行するのではなく、まず提案内容だけ見たいときに向いています。

### `sts2_execute_planned_operation`

提案されている次の一手をそのまま実行します。

## フロントエンド通知イベント

プラグインはホストのメッセージ経路を通じて、主に次の 3 系統の受動的な情報を送ります。

1. **状態・局面の同期**
   - 現在の局面要約
   - 現在の提案要約
   - 陪玩モード中の同期情報

2. **自動プレイ制御フィードバック**
   - 自動プレイを開始した
   - 一時停止した / 再開した / 停止した
   - 方針更新後に手動再開を求める通知

3. **陪玩・保護系の通知**
   - 陪玩コメント
   - リスク警告
   - 低 HP による一時停止
   - 危険攻撃による減速
   - 危険解除後の速度復帰や自動プレイ再開

これらの通知は基本的に受動配信として扱われ、メインの会話を強制的に遮らない想定です。出現頻度は次の設定にも影響されます：
- `autoplay_push_probability`
- `companion_push_probability`
- `neko_commentary_probability`
- `neko_report_hud_enabled`

## よくあるトラブル

### プラグイン入口を呼ぶと接続失敗になる

まず確認してください：

- ゲームが起動しているか。
- `STS2 AI Agent` Mod がゲームの `mods/` に正しく入っているか。
- `http://127.0.0.1:8080/health` にアクセスできるか。
- `plugin.toml` の `base_url` が正しいか。

### `http://127.0.0.1:8080/health` が開けない

優先的に確認すること：

1. ゲームが本当に起動しているか。
2. `STS2AIAgent.dll`、`STS2AIAgent.pck`、`mod_id.json` がすべてゲームの `mods/` にコピーされているか。
3. ファイル名が変更されたり、重複したり、別の場所に入っていないか。
4. 操作しているのが upstream リポジトリではなく Steam 側のゲームディレクトリか。
5. ローカルポートがファイアウォールやセキュリティソフトに塞がれていないか。

### 自動プレイは動くのに、フロントエンドに通知が来ない

次を確認してください：

- `llm_frontend_output_enabled` が `true` か。
- `llm_frontend_output_probability` が低すぎないか。
- `neko_reporting_enabled` が `true` か。
- 連携確認時はいったん `llm_frontend_output_probability` を `1` にしてみる。
- ホスト側フロントエンドが本当にプラグイン通知を受け取れているか。

### 途中のガイダンスがあまり効いていない

次を確認してください：

- 現在待機状態になっていないか。
- `sts2_send_neko_guidance` が `ok` を返しているか。
- ガイダンス内容が十分具体的か。たとえば「先に防御」「一番 HP が低い敵を狙う」「ポーションを温存する」など。
- 現在の合法アクションで、そのガイダンスを実際に満たせるか。

### 半自動タスクがなかなか終わらない

`stop_condition` を確認してください：

- `manual` / `none` の場合、自動では完了せず、`sts2_stop_autoplay` を手動で呼ぶ必要があります。
- `current_combat` の場合、そのタスク中に一度でも戦闘に入り、その後戦闘を抜けると完了します。
- `current_floor` の場合、通常は現在の階層を終えるか次の階層に入ると完了します。

`sts2_get_status` で `autoplay.task` を確認できます。

### イベント部屋・ポップアップ・遷移状態で止まる

現行版ではイベント、ポップアップ、遷移状態に対して次の優先アクションを扱えます：

- `confirm_modal`
- `dismiss_modal`
- `choose_event_option`
- `proceed`

それでも止まる場合は、まず `sts2_read_state` で現在の `screen` と `available_actions` を確認してください。

### 自動プレイが急に止まったり遅くなったりする

安全保護が発動している可能性があります：

- HP 比率が `neko_auto_low_hp_threshold` を下回ると一時停止します。
- Boss 戦や危険な攻撃では減速します。
- `neko_auto_resume_after_low_hp` が `true` の場合、HP が `neko_auto_safe_hp_threshold` まで戻ると自動再開することがあります。

`sts2_get_status` で状態を確認するか、`sts2_resume_autoplay` / `sts2_stop_autoplay` を呼んで対処してください。
