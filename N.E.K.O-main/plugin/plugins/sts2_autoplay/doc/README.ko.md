# 빠른 시작

`sts2_autoplay`는 `STS2 AI Agent`가 노출하는 로컬 *Slay the Spire 2* 상태를 N.E.K.O에 연결하기 위한 플러그인입니다. 이 플러그인은 현재 국면을 읽고, 합법적인 액션을 실행하고, 전략에 따라 자동으로 플레이하며, 네코에게 단일 카드 선택을 맡기고, 관찰 정보를 프런트엔드로 푸시하며, 백그라운드 작업 중 네코가 소프트 가이던스를 보내 다음 결정 라운드에 영향을 주도록 할 수 있습니다.

## 사용 튜토리얼

### MOD 받기

Git 사용:
```text
https://github.com/CharTyr/STS2-Agent/releases
```

### 게임 Mod 설치

Steam에서 *Slay the Spire 2* 를 우클릭한 뒤 관리 -> 로컬 파일 보기를 선택합니다.

Steam 기본 게임 디렉터리는 보통 다음과 비슷합니다:

```text
...\Steam\steamapps\common\Slay the Spire 2
```

`STS2 AI Agent` mod 를 첨탑 게임 디렉터리의 `mods/` 아래에 복사하세요.

*Slay the Spire 2* 디렉터리에 `mods` 폴더가 없다면 직접 생성하세요.

```text
mod 사용 시 세이브 데이터가 유실될 수 있습니다. 반드시 백업하거나 콘솔로 보상받으세요(첨탑 메인 메뉴에서 "~" 키를 누르고 "unlock all" 을 입력하면 모든 캐릭터와 난이도가 해금됩니다).
```

설치 후 디렉터리 구조는 다음과 비슷해야 합니다:

```text
Slay the Spire 2/
  mods/
    STS2AIAgent.dll
    STS2AIAgent.pck
    mod_id.json
```

### 게임 실행 및 인터페이스 확인

먼저 게임을 정상적으로 실행해서 Mod 가 게임과 함께 로드되도록 하세요.

처음 mod 모드로 전환할 때 한 번 튕길 수 있는데, 정상 현상입니다. 다시 게임을 실행하면 됩니다.

mod 가 로드된 뒤에는 N.E.K.O 에서 Cat Paw 를 활성화하고, 플러그인을 켜고, 플러그인 패널로 들어가 Slay the Spire 플러그인을 수동으로 시작하세요.

## 기능 개요

- 로컬 `STS2 AI Agent` HTTP 서비스에 연결해 현재 런 상황을 읽어옵니다.
- 현재 국면을 한 번에 확인할 수 있습니다. 상태를 한 번 새로고침하고, 스냅샷·국면 요약·네코 동기화 패킷을 함께 정리해 반환합니다.
- 백그라운드 자동 플레이 제어를 지원합니다. 시작·일시 중지·재개·정지뿐 아니라, 추천된 다음 한 걸음을 그대로 실행할 수도 있습니다.
- 동행 모드를 지원합니다. 메인 흐름을 끊지 않으면서 설정에 따라 관찰, 코멘트, 리마인드를 보냅니다.
- 자연어 전략 조정을 지원합니다. 사용자의 한 문장을 현재 이벤트/적 단위 선호 오버라이드로 변환해 반영합니다.
- 다음 추천 수를 미리 확인할 수 있습니다. 바로 실행하기 전에 그녀가 다음에 무엇을 하려는지 먼저 보고 판단할 수 있습니다.
- 안전 보호도 지원합니다. 저체력 일시 중지, 위험한 공격 시 감속, 위험 해소 후 속도 복구, 필요시 자동 플레이 재개 같은 동작이 포함됩니다.
- 프런트엔드로의 수동형 푸시도 지원합니다. 국면 동기화, 관찰 정보, 동행 힌트, 제어 피드백이 모두 플러그인 메시지 경로를 통해 호스트로 전달됩니다.

## 플러그인 설정

설정 파일: `plugin.toml`

| --- | --- | --- |
| `base_url` | `http://127.0.0.1:8080` | 첨탑 로컬 Agent 주소입니다. |
| `connect_timeout_seconds` | `5` | 연결 타임아웃(초)입니다. |
| `request_timeout_seconds` | `15` | 요청 타임아웃(초)입니다. |
| `poll_interval_idle_seconds` | `3` | 유휴 상태 폴링 간격입니다. |
| `poll_interval_active_seconds` | `1` | 자동 플레이 실행 중 폴링 간격입니다. |
| `action_interval_seconds` | `1.5` | 각 액션 사이에 추가로 두는 간격입니다. |
| `post_action_delay_seconds` | `0.5` | 액션 실행 후 국면이 안정되기를 기다리는 시간입니다. |

| `character_strategy` | `defect` | 기본 전략 이름입니다. 실행 중에는 현재 국면에 맞는 전략 컨텍스트로 연결됩니다. |
| `max_consecutive_errors` | `3` | 연속 오류 허용 횟수입니다. 이를 넘기면 연결 이상으로 간주합니다. |

## 일반 사용자 추천 표현

일반 사용자는 저수준 파라미터를 외울 필요가 없습니다. 우선 지금 남아 있는 고수준入口에 사용자의 말을 그대로 넘기고, 이것이 국면 확인인지, 전략 조정인지, 아니면 추천된 한 수를 실행하려는 것인지를 플러그인이 스스로 판단하게 두는 편이 좋습니다.

추천 해석 방식:

| 이렇게 말하고 싶을 때 | 잘 맞는 기능 |
| --- | --- |
| `지금 무슨 상황이야?` | `sts2_get_status` |
| `현재 국면 좀 보여줘` | `sts2_read_state` |
| `그녀가 계속 하게 해줘` / `일단 자동 플레이 멈춰줘` / `다시 계속하게 해줘` / `이제 자동으로 안 해도 돼` | autoplay 제어入口 |
| `이 이벤트는 비용이 낮은 쪽을 우선하도록 조정해줘` | `sts2_apply_user_override` |
| `다음에 뭐 하려고 해?` | `sts2_get_planned_operation` |
| `추천대로 한 수 진행해줘` | `sts2_execute_planned_operation` |
| `동행 모드 켜줘` / `동행 모드 꺼줘` | companion mode 제어入口 |

추천 흐름은 다음과 같습니다:
1. 먼저 국면을 본다
2. 다음에 그녀가 무엇을 하려는지 본다
3. 마음에 걸리는 점이 있으면 한 문장으로 방향을 조정한다
4. 마지막에 그 한 수를 그대로 둘지, 자동 플레이를 이어갈지 결정한다

## 플러그인入口

아래는 현재 메인 스크립트에서 실제로 남아 있는 대외 기능입니다. 표시 이름은 최대한 자연어로 다듬었지만, 내부 entry id 는 안정성을 위해 그대로 유지합니다.

### `sts2_health_check`

로컬 첨탑 Agent 서비스가 제대로 연결되는지 확인합니다. 연동 점검, 시작 직후 자가 점검, 오류 조사에 먼저 쓰기 좋습니다.

### `sts2_get_status`

현재 전체 상태를 확인합니다. 연결이 정상인지, 지금 어떤 화면인지, 자동 플레이가 돌고 있는지, standby 인지, 최근 오류와 현재 모드가 어떤지 등을 한 번에 볼 수 있습니다.

### `sts2_read_state`

현재 국면을 한 번 새로고침하고, 다음 세 층의 정보를 함께 돌려줍니다:
- 현재 스냅샷
- 현재 상황 요약
- 현재 네코 동기화 패킷

다음 행동을 결정하기 전에 전체 상태를 한 번에 보고 싶을 때 적합합니다.

### `sts2_set_standby`

대기 모드를 전환합니다. 대기 중에는 액션 실행을 멈추지만, 상태 정리와 동기화 준비 기능은 유지됩니다.

### `sts2_start_autoplay`

그녀가 계속 플레이하게 둡니다. 백그라운드 자동 플레이를 켜서 현재 국면을 그대로 진행합니다.

### `sts2_pause_autoplay`

일단 자동 플레이를 멈춥니다. 직접 손을 대고 싶을 때나, 다음 행동 전에 전략을 바꾸고 싶을 때 유용합니다.

### `sts2_resume_autoplay`

멈춰 둔 자동 플레이를 그대로 이어서 진행합니다.

### `sts2_stop_autoplay`

이제 자동으로 플레이하지 않게 합니다. 백그라운드 자동 플레이를 완전히 멈추고 조작권을 되돌립니다.

### `sts2_enable_companion_mode`

동행 모드를 켭니다. 국면 정리와 관찰 푸시가 더 적극적으로 이뤄지고, 적절한 순간에 코멘트와 경고도 들어옵니다.

### `sts2_disable_companion_mode`

동행 모드를 끕니다. 동행 코멘트 층만 끄고, 기본 상태 읽기와 자동 플레이 제어는 그대로 사용할 수 있습니다.

### `sts2_apply_user_override`

사용자의 한 문장으로 전략을 조정합니다. 현재 장면에 비춰, 그 문장을 이벤트 단위 또는 적 단위의 선호 오버라이드로 바꿉니다.

이入口에는 추가적인 안전 동작도 있습니다:
- 자동 플레이가 실행 중이면 **먼저 자동 플레이를 일시 중지합니다**
- 전략을 갱신한 뒤에는 **계속하려면 자동 플레이를 수동으로 재개하라고 안내합니다**
- 명시적으로 재개되기 전에는 그 상태로 멋대로 계속 진행하지 않습니다

### `sts2_get_planned_operation`

다음에 무엇을 하려는지 확인합니다. 바로 실행하기보다, 먼저 제안 내용을 보고 싶을 때 적합합니다.

### `sts2_execute_planned_operation`

추천된 다음 한 수를 그대로 실행합니다.

## 프런트엔드 푸시 이벤트

플러그인은 호스트 메시지 경로를 통해 주로 다음 세 계열의 수동형 정보를 보냅니다.

1. **상태 및 국면 동기화**
   - 현재 국면 요약
   - 현재 제안 요약
   - 동행 모드 중의 동기화 정보

2. **자동 플레이 제어 피드백**
   - 자동 플레이 시작
   - 일시 중지 / 재개 / 정지
   - 전략 갱신 후 수동 재개를 요구하는 안내

3. **동행 및 보호성 알림**
   - 동행 코멘트
   - 위험 리마인드
   - 저체력 일시 중지
   - 위험 공격 감속
   - 위험 해소 후 속도 복구 또는 자동 플레이 재개

이 푸시들은 기본적으로 수동 전달 의미를 가지며, 메인 대화를 강제로 끊지 않도록 설계되어 있습니다. 실제 발생 빈도는 다음 설정의 영향도 받습니다:
- `autoplay_push_probability`
- `companion_push_probability`
- `neko_commentary_probability`
- `neko_report_hud_enabled`

## 자주 있는 문제 해결

### 플러그인入口 호출 시 연결 실패가 뜬다

먼저 다음을 확인하세요:

- 게임이 이미 실행 중인지
- `STS2 AI Agent` Mod 가 게임의 `mods/` 폴더에 올바르게 들어 있는지
- `http://127.0.0.1:8080/health` 에 접속 가능한지
- `plugin.toml` 의 `base_url` 이 올바른지

### `http://127.0.0.1:8080/health` 가 열리지 않는다

다음 순서로 확인하세요:

1. 게임이 정말 실행 중인지
2. `STS2AIAgent.dll`, `STS2AIAgent.pck`, `mod_id.json` 이 모두 게임의 `mods/` 에 복사되어 있는지
3. 파일명이 바뀌었거나 중복되었거나, 잘못된 폴더에 들어 있지 않은지
4. 지금 작업 중인 위치가 upstream 저장소가 아니라 Steam 쪽 게임 디렉터리인지
5. 로컬 포트를 방화벽이나 보안 소프트웨어가 막고 있지 않은지

### 자동 플레이는 도는데 프런트엔드에 메시지가 안 온다

다음을 확인하세요:

- `llm_frontend_output_enabled` 가 `true` 인지
- `llm_frontend_output_probability` 가 너무 낮지 않은지
- `neko_reporting_enabled` 가 `true` 인지
- 연동 확인 중이라면 `llm_frontend_output_probability` 를 잠시 `1` 로 올려 보기
- 호스트 프런트엔드가 실제로 플러그인 푸시 메시지를 받고 있는지

### 중간 가이던스가 잘 먹지 않는 것 같다

다음을 확인하세요:

- 현재 대기 상태에 들어가 있지 않은지
- `sts2_send_neko_guidance` 가 `ok` 를 반환했는지
- 가이던스가 충분히 구체적인지. 예: `먼저 방어`, `HP 가장 낮은 적부터`, `포션 아끼기`
- 현재 합법 액션으로 그 가이던스를 실제로 만족시킬 수 있는지

### 반자동 작업이 끝나지 않는다

`stop_condition` 을 확인하세요:

- `manual` / `none` 인 경우 자동 완료되지 않으므로 `sts2_stop_autoplay` 를 직접 호출해야 합니다.
- `current_combat` 인 경우 작업 중 한 번이라도 전투에 들어간 뒤, 전투에서 빠져나오면 완료됩니다.
- `current_floor` 인 경우 보통 현재 층을 끝내거나 다음 층에 들어가면 완료됩니다.

`sts2_get_status` 로 `autoplay.task` 를 확인할 수 있습니다.

### 이벤트 방, 팝업, 전환 상태에서 멈춘다

현재 버전은 이벤트, 팝업, 전환 상태에서 다음 우선 액션을 처리할 수 있습니다:

- `confirm_modal`
- `dismiss_modal`
- `choose_event_option`
- `proceed`

그래도 멈춘다면 먼저 `sts2_read_state` 로 현재 `screen` 과 `available_actions` 를 확인하세요.

### 자동 플레이가 갑자기 멈추거나 느려진다

안전 보호가 발동했을 수 있습니다:

- HP 비율이 `neko_auto_low_hp_threshold` 아래로 내려가면 일시 중지됩니다.
- Boss 전이나 위험한 공격 때는 감속합니다.
- `neko_auto_resume_after_low_hp` 가 `true` 이면 HP 가 `neko_auto_safe_hp_threshold` 까지 돌아왔을 때 자동 재개될 수 있습니다.

`sts2_get_status` 로 상태를 확인하거나, `sts2_resume_autoplay` / `sts2_stop_autoplay` 를 호출해 처리하세요.
